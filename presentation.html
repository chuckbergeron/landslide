Your Very Own Rails

https://github.com/rails/rails => FORK IT! => https://github.com/chuckbergeron/rails

 1 $ take ~/Git/rails-dev-box
 2 $ git clone git://github.com/chuckbergeron/rails.git rails
 3 $ cd rails
 4 
 5 # Add the original rails repo as a remote source...
 6 $ git remote add upstream git://github.com/rails/rails.git
 7 $ git fetch upstream
 8 $ git checkout master
 9 
10 # ...so you can stay up-to-date:
11 $ git rebase upstream/master

Fake It 'Til Ya Make It

Create a fake application to test Rails against:

 1 # If you RVM...
 2 $ rvm use ruby-2.0.0-p0@rails
 3 
 4 $ cd ~/Git/rails-dev-box/rails
 5 $ bundle install
 6 
 7 # Use the Rails 4 binary to create your new app (needs --dev flag)
 8 $ railties/bin/rails new ../test_app --dev
 9 
10 gem 'rails', path: '/Users/charlesbergeron/Git/rails-dev-box/rails'

Testing

Develop locally (on the host, using your vim, sublime, what-have-you)

Test in the VM:

 1 host $ cd ~/Git/rails-dev-box
 2 host $ vagrant ssh
 3 
 4 remote $ cd /vagrant/rails
 5 remote $ bundle exec rake
 6 
 7 ### Stop! Don't test the whole thing!
 8 
 9 remote $ cd /activemodel
10 remote $ bundle exec rake
11 
12 # When you're done, cleanup after yourself...
13 remote $ exit
14 host $ vagrant halt
